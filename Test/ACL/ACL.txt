한 회사에서는 이미지 처리를 위해 2계층 애플리케이션을 운영하고 있습니다. 
애플리케이션은 각각 퍼블릭 서브넷 1개와 프라이빗 서브넷 1개가 있는 2개의 가용 영역을 사용합니다. 
웹 계층용 ALB(Application Load Balancer)는 퍼블릭 서브넷을 사용합니다. 애플리케이션 계층의 Amazon EC2 인스턴스는 프라이빗 서브넷을 사용합니다.
사용자들은 애플리케이션이 예상보다 느리게 실행되고 있다고 보고합니다. 
웹 서버 로그 파일의 보안 감사에 따르면 애플리케이션이 소수의 IP 주소로부터 수백만 건의 불법적인 요청을 받고 있는 것으로 나타났습니다. 
솔루션 설계자는 즉각적인 성능 문제를 해결해야 하며 회사는 보다 영구적인 솔루션을 조사해야 합니다.
이 요구 사항을 충족하기 위해 솔루션 설계자는 무엇을 권장해야 합니까?

A. 애플리케이션 계층 서브넷에 대한 네트워크 ACL을 수정합니다. 리소스를 소비하는 IP 주소에 대한 인바운드 거부 규칙을 추가합니다.
B. 웹 계층 서브넷에 대한 네트워크 ACL을 수정합니다. 리소스를 소비하는 IP 주소에 대한 인바운드 거부 규칙을 추가합니다.
C. 웹 계층에 대한 인바운드 보안 그룹을 수정합니다. 리소스를 소비하는 IP 주소에 대한 거부 규칙을 추가합니다.
D. 애플리케이션 계층에 대한 인바운드 보안 그룹을 수정합니다. 리소스를 소비하는 IP 주소에 대한 거부 규칙을 추가합니다.

정답: B. 웹 계층 서브넷에 대한 네트워크 ACL을 수정합니다. 리소스를 소비하는 IP 주소에 대한 인바운드 거부 규칙을 추가합니다.
주요 키워드 설명:
네트워크 ACL (Access Control List): 네트워크 ACL은 VPC 내의 서브넷을 통해 들어오고 나가는 트래픽을 허용하거나 거부하는 데 사용되는 선택적인 레이어입니다. 
각 네트워크 ACL은 여러 규칙을 가질 수 있으며, 이 규칙들은 특정 IP 주소 범위로부터의 트래픽을 허용하거나 거부할 수 있습니다.

웹 계층 서브넷: 웹 계층 서브넷은 애플리케이션의 웹 서버 또는 웹 애플리케이션 로드 밸런서와 같은 웹 계층 리소스를 호스팅하는 데 사용되는 퍼블릭 서브넷입니다.

솔루션의 이점:
즉각적인 성능 개선: 웹 계층 서브넷의 네트워크 ACL을 수정하여 리소스를 과도하게 소비하는 IP 주소로부터의 인바운드 요청을 거부함으로써, 불법적인 트래픽을 초기 단계에서 차단할 수 있습니다. 
이는 웹 서버에 대한 부담을 줄이고 애플리케이션의 성능을 즉각적으로 개선할 수 있는 방법입니다.

영구적인 솔루션 조사: 이 조치는 단기적인 해결책으로서의 역할을 하며, 회사가 보다 영구적인 솔루션을 찾는 동안 임시적인 성능 문제를 해결합니다. 
장기적으로는 보안 정책을 강화하고, 고급 보안 서비스를 도입하여 애플리케이션을 보호하는 방안을 고려할 수 있습니다.

이 솔루션은 웹 계층에서 불법적인 요청을 차단하여 애플리케이션의 성능 문제를 해결하고, 애플리케이션과 데이터베이스 계층의 보안을 강화하는 데 도움을 줍니다.
