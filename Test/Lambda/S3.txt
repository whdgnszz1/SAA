한 회사는 사용자가 사진을 업로드하고 이미지에 액자를 추가할 수 있는 이미지 분석 애플리케이션을 만들었습니다. 
사용자는 이미지와 메타데이터를 업로드하여 이미지에 추가할 사진 프레임을 나타냅니다. 
애플리케이션은 단일 Amazon EC2 인스턴스와 Amazon DynamoDB를 사용하여 메타데이터를 저장합니다.
애플리케이션이 점점 인기를 얻고 있으며 사용자 수가 증가하고 있습니다. 회사에서는 시간대와 요일에 따라 동시접속자 수가 크게 달라질 것으로 예상하고 있다. 
회사는 늘어나는 사용자 기반의 요구 사항을 충족하도록 애플리케이션을 확장할 수 있는지 확인해야 합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

A. AWS Lambda를 사용하여 사진을 처리합니다. DynamoDB에 사진과 메타데이터를 저장합니다.
B. Amazon Kinesis Data Firehose를 사용하여 사진을 처리하고 사진과 메타데이터를 저장합니다.
C. AWS Lambda를 사용하여 사진을 처리합니다. Amazon S3에 사진을 저장합니다. 메타데이터를 저장하기 위해 DynamoDB를 보관합니다.
D. EC2 인스턴스 수를 3개로 늘립니다. 프로비저닝된 IOPS SSD(io2) Amazon Elastic Block Store(Amazon EBS) 볼륨을 사용하여 사진과 메타데이터를 저장합니다.

정답: C. AWS Lambda를 사용하여 사진을 처리합니다. Amazon S3에 사진을 저장합니다. 메타데이터를 저장하기 위해 DynamoDB를 보관합니다.

주요 키워드 설명:
AWS Lambda: 서버를 프로비저닝하거나 관리하지 않고도 코드를 실행할 수 있는 이벤트 기반, 서버리스 컴퓨팅 서비스입니다. 
Lambda는 요구 사항에 따라 자동으로 확장되므로 트래픽의 변동에 효과적으로 대응할 수 있습니다.

Amazon S3 (Simple Storage Service): 인터넷 스토리지 서비스로, 데이터를 안전하게 저장하고 전 세계 어디서나 접근할 수 있도록 지원합니다.
S3는 높은 내구성과 가용성을 제공하며, 대규모 데이터를 저장하기에 적합합니다.

Amazon DynamoDB: 완전관리형 NoSQL 데이터베이스 서비스로, 빠르고 일관된 성능 및 자동 확장 기능을 제공합니다. 
메타데이터와 같은 구조화된 데이터를 효율적으로 저장하고 검색하는 데 적합합니다.

솔루션의 이점:
확장성: AWS Lambda는 처리해야 할 사진의 양에 따라 자동으로 확장되므로, 애플리케이션의 요구 사항을 충족할 수 있는 충분한 컴퓨팅 리소스를 제공합니다.

성능과 가용성: Amazon S3는 사진 데이터를 안정적으로 저장하며, DynamoDB는 메타데이터의 빠른 읽기 및 쓰기를 보장합니다.

비용 효율성: 서버리스 아키텍처는 사용한 만큼만 비용을 지불하므로, EC2 인스턴스와 EBS 볼륨을 직접 관리하는 것보다 비용을 절감할 수 있습니다.

왜 다른 옵션들은 적합하지 않은가?
A 옵션: DynamoDB에 사진을 저장하는 것은 권장되지 않습니다. 사진과 같은 대용량 바이너리 데이터는 S3에 저장하는 것이 더 효율적입니다.

B 옵션: Amazon Kinesis Data Firehose는 주로 스트리밍 데이터를 처리하고 저장하는 데 사용되며, 사진 처리와 같은 작업에는 적합하지 않습니다.

D 옵션: EC2 인스턴스 수를 늘리고 IOPS SSD 볼륨을 사용하는 것은 확장성과 비용 효율성 측면에서 최선의 해결책이 아닙니다. 트래픽 변동에 자동으로 대응하기 어렵고, 관리가 복잡하며 비용이 더 듭니다.

C 옵션은 트래픽의 변동성을 효과적으로 처리하고, 확장성을 제공하며, 사진과 메타데이터를 효율적으로 관리할 수 있는 솔루션을 제공합니다.
