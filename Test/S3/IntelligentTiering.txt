솔루션 설계자는 Amazon S3를 사용하여 새로운 디지털 미디어 애플리케이션의 스토리지 아키텍처를 설계하고 있습니다. 
미디어 파일은 가용 영역 손실에 대해 복원력이 있어야 합니다. 일부 파일은 자주 액세스되는 반면 다른 파일은 예측할 수 없는 패턴으로 거의 액세스되지 않습니다. 
솔루션 설계자는 미디어 파일 저장 및 검색 비용을 최소화해야 합니다. 이러한 요구 사항을 충족하는 스토리지 옵션은 무엇입니까?

A. S3 표준
B. S3 지능형 계층화
C. S3 표준-Infrequent Access(S3 표준-IA)
D. S3 One Zone-Infrequent Access(S3 One Zone-IA)

정답: B. S3 지능형 계층화
주요 키워드 설명:
S3 지능형 계층화 (S3 Intelligent-Tiering): 이 스토리지 클래스는 액세스 패턴이 불규칙하거나 예측할 수 없는 데이터를 위해 설계되었습니다. 
S3 지능형 계층화는 사용 패턴을 모니터링하고, 데이터를 자동으로 자주 액세스되는 계층(frequent access tier)과 거의 액세스되지 않는 계층(infrequent access tier)으로 이동시켜 저장 비용을 최적화합니다.

스토리지 옵션 비교:
A. S3 표준: 일반적으로 자주 액세스되는 데이터를 위한 스토리지 클래스입니다. 모든 S3 데이터는 기본적으로 세 가용 영역에 걸쳐 복제되어 고가용성과 내구성을 제공합니다. 
그러나 모든 데이터를 동일한 비용으로 저장하기 때문에 비용 최적화 측면에서 가장 효율적이지 않을 수 있습니다.

C. S3 표준-Infrequent Access (S3 표준-IA): 거의 액세스되지 않지만, 필요할 때 빠르게 액세스할 수 있어야 하는 데이터를 위해 설계되었습니다. 
저장 비용은 S3 표준보다 낮지만, 데이터 검색 비용이 발생합니다.

D. S3 One Zone-Infrequent Access (S3 One Zone-IA): S3 표준-IA와 유사하지만, 데이터를 단일 가용 영역에만 저장합니다. 가용 영역 손실 시 데이터 복원력이 필요한 경우 적합하지 않습니다.

솔루션의 이점:
S3 지능형 계층화를 사용하면 미디어 파일을 자동으로 가장 비용 효율적인 방식으로 관리할 수 있습니다. 
자주 액세스되는 파일은 빠른 액세스를 위해 보관되고, 거의 액세스되지 않는 파일은 비용을 절감하기 위해 저렴한 계층으로 이동됩니다. 
이 접근 방식은 미디어 파일 저장 및 검색 비용을 최소화하면서도 애플리케이션의 요구 사항을 충족합니다.
