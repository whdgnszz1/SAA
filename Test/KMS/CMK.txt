한 회사가 Amazon RDS DB 인스턴스에 데이터를 저장할 계획입니다. 회사는 저장 데이터를 암호화해야 합니다.
이 요구 사항을 충족하려면 솔루션 설계자가 무엇을 해야 합니까?

A. 암호화 키를 생성하고 AWS Secrets Manager에 키를 저장합니다. 키를 사용하여 DB 인스턴스를 암호화합니다.
B. AWS Certificate Manager(ACM)에서 인증서를 생성합니다. 인증서를 사용하여 DB 인스턴스에서 SSL/TLS를 활성화합니다.
C. AWS Key Management Service(AWS KMS)에서 고객 마스터 키(CMK)를 생성합니다. DB 인스턴스에 대한 암호화를 활성화합니다.
D. AWS Identity and Access Management {IAM)에서 인증서를 생성합니다. 인증서를 사용하여 DB 인스턴스에서 SSUTLS를 활성화합니다.

회사가 Amazon RDS DB 인스턴스에 저장할 데이터를 암호화하려는 시나리오에 관한 질문입니다. 
이 요구 사항을 충족하기 위해 솔루션 설계자가 수행해야 하는 작업에 대해 설명하고 있습니다. 
정답은 C. AWS Key Management Service(AWS KMS)에서 고객 마스터 키(CMK)를 생성합니다. DB 인스턴스에 대한 암호화를 활성화합니다.

주요 키워드 설명:
Amazon RDS: Amazon Relational Database Service(RDS)는 클라우드에서 관계형 데이터베이스를 쉽게 설정, 운영, 확장할 수 있는 관리형 서비스입니다. 
사용자는 몇 분 내에 인프라 프로비저닝 없이 데이터베이스 인스턴스를 배포할 수 있으며, 관리 작업의 대부분이 자동화됩니다.

AWS Key Management Service (AWS KMS): AWS KMS는 암호화 키를 쉽게 생성하고 관리할 수 있게 해주는 서비스입니다. 
이를 통해 데이터를 보호하고, 키 사용을 감사할 수 있으며, AWS 서비스 및 애플리케이션 내에서 암호화 키를 사용할 수 있습니다.

고객 마스터 키 (CMK): CMK는 AWS KMS에서 사용자가 생성하거나 AWS가 사용자를 대신해 생성하는 암호화 키입니다. 이 키는 데이터를 암호화하고 복호화하는 데 사용됩니다.

정답 설명:
C 옵션은 Amazon RDS 데이터베이스 인스턴스에 저장된 데이터를 암호화하는 가장 직접적이고 효과적인 방법을 제공합니다. 
AWS KMS에서 CMK를 생성하고 이 키를 사용하여 DB 인스턴스의 암호화를 활성화함으로써, 데이터베이스 인스턴스, 자동 백업, 읽기 복제본, 스냅샷을 포함한 기본 스토리지가 암호화됩니다.

오답 설명:

A 옵션은 AWS Secrets Manager를 언급하지만, 이는 암호화 키를 저장하는 용도로 사용되며, RDS 인스턴스의 데이터를 직접 암호화하는 데 사용되지 않습니다.

B 및 D 옵션은 SSL/TLS와 IAM 인증서를 언급합니다. 이는 데이터를 전송 중에 암호화하는 데 사용되지만, RDS 인스턴스에 저장된 데이터를 암호화하는 데는 적합하지 않습니다.
결론적으로, AWS KMS를 사용하여 고객 마스터 키를 생성하고 RDS DB 인스턴스에 대한 암호화를 활성화하는 것은 Amazon RDS에 저장된 데이터를 암호화하기 위한 가장 적절한 방법입니다.
