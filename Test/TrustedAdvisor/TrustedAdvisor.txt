회사에 통합 결제를 사용하는 여러 AWS 계정이 있습니다. 이 회사는 Oracle On-Demand DB 인스턴스용 활성 고성능 Amazon RDS를 90일 동안 여러 개 실행합니다. 
회사의 재무팀은 통합 결제 계정 및 기타 모든 AWS 계정에서 AWS Trusted Advisor에 액세스할 수 있습니다. 
재무팀은 RDS에 대한 Trusted Advisor 확인 권장 사항에 액세스하려면 적절한 AWS 계정을 사용해야 합니다. 
재무팀은 RDS 비용을 줄이기 위해 적절한 Trusted Advisor 점검을 검토해야 합니다.
이러한 요구 사항을 충족하기 위해 재무팀은 어떤 단계 조합을 취해야 합니까? (2개를 선택하세요.)

A. RDS 인스턴스가 실행 중인 계정의 Trusted Advisor 권장 사항을 사용합니다.
B. 통합 결제 계정의 Trusted Advisor 권장 사항을 사용하여 모든 RDS 인스턴스 확인을 동시에 확인하세요.
C. Amazon RDS 예약 인스턴스 최적화에 대한 Trusted Advisor 점검을 검토합니다. 
D. Amazon RDS 유휴 DB 인스턴스에 대한 Trusted Advisor 점검을 검토합니다.
E. Amazon Redshift 예약 노드 최적화에 대한 Trusted Advisor 점검을 검토합니다. 

정답:
B. 통합 결제 계정의 Trusted Advisor 권장 사항을 사용하여 모든 RDS 인스턴스 확인을 동시에 확인하세요.
C. Amazon RDS 예약 인스턴스 최적화에 대한 Trusted Advisor 점검을 검토합니다.
주요 키워드 설명:
Trusted Advisor: AWS Trusted Advisor는 AWS 리소스와 계정을 최적화하여 성능과 보안을 개선하고, 비용을 절감하는 데 도움을 주는 도구입니다. 
Trusted Advisor는 여러 카테고리에 걸쳐 권장 사항을 제공합니다.

통합 결제 계정: 여러 AWS 계정의 비용을 하나의 결제 계정에 집계하는 AWS Organizations의 기능입니다. 
통합 결제 계정을 사용하면 모든 계정의 비용과 사용량을 중앙에서 관리하고 분석할 수 있습니다.

Amazon RDS 예약 인스턴스 최적화: 예약 인스턴스는 사전 결제를 통해 할인된 요금으로 RDS 인스턴스를 사용할 수 있게 해주는 결제 옵션입니다. 
Trusted Advisor는 사용 패턴에 기반하여 예약 인스턴스 구매를 권장함으로써 비용 절감을 돕습니다.

솔루션의 이점:
B 옵션: 통합 결제 계정을 통해 모든 RDS 인스턴스에 대한 Trusted Advisor 권장 사항을 한 번에 확인할 수 있으므로, 재무팀은 모든 계정에 걸쳐 비용 절감 기회를 쉽게 파악할 수 있습니다.

C 옵션: Amazon RDS 예약 인스턴스 최적화 점검을 검토함으로써, 재무팀은 장기적인 비용 절감을 위해 예약 인스턴스를 구매할 수 있는 인스턴스를 식별할 수 있습니다.

왜 다른 옵션들은 적합하지 않은가?
A, D, E 옵션: 이 옵션들은 특정 인스턴스나 다른 AWS 서비스에 대한 점검을 포함할 수 있지만, 모든 RDS 인스턴스의 비용 최적화에 집중하지 않거나 통합 결제 계정의 전체적인 관점을 제공하지 않습니다.
정답 B와 C는 재무팀이 통합 결제 계정과 연결된 모든 AWS 계정에서 RDS 비용을 효과적으로 관리하고 최적화할 수 있게 해주는 가장 효율적인 접근 방식을 제공합니다.
