회사는 디지털 미디어 스트리밍 애플리케이션을 호스팅하기 위해 Amazon Elastic Kubernetes Service(Amazon EKS) 클러스터를 생성해야 합니다. 
EKS 클러스터는 저장을 위해 Amazon Elastic Block Store(Amazon EBS) 볼륨이 지원하는 관리형 노드 그룹을 사용합니다. 
회사는 AWS Key Management Service(AWS KMS)에 저장된 고객 관리형 키를 사용하여 모든 저장 데이터를 암호화해야 합니다. 
최소한의 운영 오버헤드로 이 요구 사항을 충족할 수 있는 작업 조합은 무엇입니까? (2개를 선택하세요.)

A. 고객 관리 키를 사용하여 데이터 암호화를 수행하는 Kubernetes 플러그인을 사용합니다. 
B. EKS 클러스터를 생성한 후 EBS 볼륨을 찾습니다. 고객 관리형 키를 사용하여 암호화를 활성화합니다.
C. EKS 클러스터가 생성될 AWS 리전에서 기본적으로 EBS 암호화를 활성화합니다. 고객 관리형 키를 기본 키로 선택합니다.
D. EKS 클러스터 생성 고객 관리형 키에 권한을 부여하는 권한이 있는 IAM 역할을 생성합니다. 역할을 EKS 클러스터와 연결합니다.
E. 고객 관리 키를 EKS 클러스터에 Kubernetes 비밀로 저장합니다. 고객 관리형 키를 사용하여 EBS 볼륨을 암호화합니다.

정답: 
C. EKS 클러스터가 생성될 AWS 리전에서 기본적으로 EBS 암호화를 활성화합니다. 고객 관리형 키를 기본 키로 선택합니다.
D. EKS 클러스터 생성 고객 관리형 키에 권한을 부여하는 권한이 있는 IAM 역할을 생성합니다. 역할을 EKS 클러스터와 연결합니다.

주요 키워드 설명:
EBS 암호화 기본 활성화: AWS 리전에서 새로 생성되는 모든 EBS 볼륨과 스냅샷에 대해 자동 암호화를 활성화하는 기능입니다. 
사용자는 서비스 관리 키 또는 고객 관리형 키 중 하나를 선택하여 암호화에 사용할 수 있습니다.

고객 관리형 키: AWS KMS에 저장된 키로, 고객이 직접 관리할 수 있습니다. 
이 키는 데이터 암호화 또는 다른 AWS 서비스와의 통합에 사용될 수 있습니다.

IAM 역할과 연결: AWS 리소스에 대한 액세스 권한을 제어하는 IAM 역할을 생성하고, 
이 역할을 EKS 클러스터에 연결하여 클러스터가 고객 관리형 키에 액세스할 수 있도록 합니다.

솔루션의 이점:
C 옵션: AWS 리전에서 EBS 암호화를 기본적으로 활성화하고 고객 관리형 키를 기본 키로 설정함으로써, 모든 새로운 EBS 볼륨과 스냅샷이 자동으로 해당 키로 암호화됩니다. 
이는 EKS 클러스터와 관련된 모든 스토리지 리소스에 대한 데이터 보호를 강화합니다.

D 옵션: IAM 역할을 생성하고 이를 EKS 클러스터와 연결하여, 클러스터가 고객 관리형 키에 액세스하여 Kubernetes 시크릿과 같은 리소스를 암호화할 수 있도록 합니다. 
이는 클러스터 내에서 사용되는 민감한 데이터에 대한 추가 보안 계층을 제공합니다.

왜 다른 옵션들은 적합하지 않은가?
A 옵션과 E 옵션: Kubernetes 플러그인이나 EKS 클러스터에 Kubernetes 비밀로 키를 저장하는 방식은 이 시나리오에서 요구하는 저장 데이터 암호화를 직접적으로 지원하지 않습니다.

B 옵션: Amazon S3 버킷을 사용하는 것은 이 시나리오와 관련이 없으며, EBS 볼륨 저장 데이터 암호화에 대한 요구 사항을 충족시키지 않습니다.

정답 C와 D는 Amazon EKS 클러스터와 Amazon EBS 볼륨을 사용하여 디지털 미디어 스트리밍 애플리케이션을 호스팅하면서 모든 저장 데이터를 고객 관리형 키로 암호화하는 요구 사항에 적합합니다.
